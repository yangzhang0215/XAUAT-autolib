# 研讨室 `roomId` 对照表

以下 `roomId` 来自 `python/runtime/seminar-tool-discover-20260407.json` 顶层 `rooms`，这是提交预约时应该使用的真实房间 ID。

说得直白一点：

- 真正要传给预约接口的是顶层 `rooms[*].roomId`
- 不要把 `roomDetail[roomId].applyId` 当成另一个房间 ID，它更像“这组申请配置的 ID”
- 不要把 `roomDetail[roomId].children[*].id` 当成房间 ID，那是分类下面的枚举项或分组选项
- 同一个研讨室在这份导出里没有出现“一间房对应多个真实 `roomId`”的情况

如果你要指定房间预约，直接传这里的 `roomId` 就行，例如：

```powershell
py -3 python\seminar_cli.py reserve --room-id 69 --participant 2504000001 --participant 2504000002 --wait
```
