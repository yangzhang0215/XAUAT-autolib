# 独立研讨室预约 GUI 使用说明

这套 GUI 是我专门给研讨室预约单独做的，不走统一 CLI。

如果你只是想把研讨室约起来，直接用它就行。

## 下载和启动

如果你是从 GitHub Release 下载的，我建议优先用：

- `xauat-seminar-gui-portable.zip`

下载后：

1. 解压压缩包
2. 进入解压目录
3. 双击 `xauat-seminar-gui.exe`

这份是免 Python 环境的，正常情况下不需要额外装 Python。

## 界面结构

GUI 现在分成三块：

- `仪表盘（预约）`：看当天房间、执行预约
- `偏好设置`：填账号、时间、成员、房间优先级
- `运行日志`：看实时日志和历史日志

右下角还有日志控制台，展开后可以直接看当前执行过程。

## 第一次使用该填什么

先去“偏好设置”，把这些内容填好：

- 统一身份认证账号
- 统一身份认证密码
- 手机号
- 触发时间
- 开始时间
- 结束时间
- 参与人学号
- `roomId` 优先顺序
- 主题
- 内容
- 是否公开

填完后点保存。

程序会把配置写到 exe 同目录下：

- `seminar.config.local.json`

## 每个字段怎么填

### 账号和密码

这里填的是主账号，也就是统一身份认证账号和密码。

研讨室预约只需要主账号登录，其他成员不需要再登录。

### 手机号

填预约表单里要提交的手机号。

### 触发时间

默认是：

- `08:00:00`

这表示程序会在这个时间点开始发预约请求。

### 开始时间和结束时间

这里只填当天的预约时间，不需要填预约日期。

例如：

- `08:00`
- `16:15`

程序只支持预约当天，结束时间不能超过：

- `22:30`

### 参与人学号

一行一个，或者用空格、逗号分开都行。

例如：

```text
2504000001
2504000002
```

不要把主账号本人再填进参与人里。

### roomId 优先顺序

这个是自动选房的核心配置，一行一个，按优先顺序填写。

例如：

```text
69
70
71
```

程序会按这个顺序去尝试，命中第一间可预约房间就提交。

如果你只想约固定某一间房，就只填一个 `roomId`。

房间对照表：

- [研讨室 roomId 对照表](seminar-room-id-table.md)

### 主题、内容、是否公开

这几项就是网页预约时表单里要填的内容，GUI 会原样带上去提交。

## 主界面按钮怎么用

### 获取今日空闲

点击后会：

- 拉取当天房间数据
- 按楼层在主界面展示
- 同时导出 JSON 和 TXT

导出文件位置：

- `runtime\seminar-tool-discover-YYYYMMDD.json`
- `runtime\seminar-tool-discover-YYYYMMDD.txt`

TXT 比较适合快速看。

### 立即预约

适合已经到开放时间、现在就想提交的时候使用。

点击后会立刻按当前配置发起预约。

### 到点预约

适合你提前把配置填好，然后等程序自动到点提交。

点击后程序会一直等待到触发时间，然后自动发请求。

如果你点击的时候，今天已经过了触发时间，程序会自动切到第二天同一时间执行。

## 到点预约时电脑要保持什么状态

这个地方很重要。

可以：

- 锁屏
- 关闭显示器

不可以：

- 睡眠
- 休眠
- 关机
- 退出 GUI
- 断网

一句话总结就是：

- 锁屏可以
- 睡眠不行

## 预约时间规则

- 只支持预约当天
- 每天 `08:00:00` 开始预约
- 图书馆 `22:30` 关门
- 单次预约最长 4 小时
- 超过 4 小时会自动拆成两段
- 两段之间固定间隔 15 分钟
- 总跨度不能超过 8 小时 15 分钟
- 两段预约请求之间会间隔 10 秒发送

比如你填：

- `08:00` 到 `16:15`

程序会自动按下面两段提交：

- `08:00-12:00`
- `12:15-16:15`

## 房间状态说明

主界面房间列表里的状态我现在做得比较直白：

- `可自动尝试`：今天至少还有一段满足条件的时间
- `无法预约`：剩下的空档已经不够用
- `当天不可约`：当天本来就不在可预约日期里
- `需上传材料`：这类房间不走自动预约

## 自动选房逻辑

自动选房不是“评分制”，就是最简单的顺序命中：

1. 先按你填写的 `roomId` 顺序取候选
2. 逐个检查当天是否可约
3. 再检查时间窗口、人数、禁用时段、上传材料要求等
4. 命中第一间符合条件的房间就提交

如果前面的房间不行，会继续试下一间。

## 运行时会生成哪些文件

程序运行后，同目录下会出现这些内容：

- `seminar.config.local.json`：你的 GUI 配置
- `runtime\state.json`：最近一次执行状态
- `runtime\logs\YYYY-MM-DD.jsonl`：当天日志
- `runtime\seminar-tool-discover-YYYYMMDD.json`
- `runtime\seminar-tool-discover-YYYYMMDD.txt`

## 常见问题

### 1. 为什么只需要主账号登录

因为程序真正登录的是主账号，其他成员只是通过学号去解析成员信息，不需要再登录副账号。

### 2. 为什么同一个研讨室看起来像有不同 id

真正提交预约时要传的是房间自己的 `roomId`。你在别的接口里看到的申请 id、分组 id、子项 id，不是提交预约时该传的房间 id。

### 3. 如果没有本地账号信息怎么办

直接打开 GUI，在设置页填账号和密码，保存后就能用。

### 4. 如果第一段成功、第二段失败会怎样

这时候程序会把结果记成部分成功，你可以在日志里看到具体是哪一段失败。

### 5. 想看当天空闲但不想一间间点怎么办

先点“获取今日空闲”，然后直接打开生成的 TXT 摘要。
